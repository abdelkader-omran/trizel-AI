# TRIZEL Canonical Metadata Templates

## Overview

This directory contains the canonical metadata templates used across TRIZEL publications and research outputs.

## Files

### 1. CANONICAL_METADATA_TEMPLATE.yaml
**Status:** Single Source of Truth

This YAML file is the **authoritative template** for all author and publication metadata across TRIZEL repositories. It defines the standard schema and default values for:
- Author information (name, ORCID, affiliation)
- Publication metadata (title, description, creators, contributors)
- Licensing and access rights
- Keywords and related identifiers
- Zenodo/DataCite compatible fields

### 2. CANONICAL_METADATA_TEMPLATE.json
**Status:** Mirror (Derived)

This JSON file is a **faithful mirror** of the YAML template, provided for consumers and systems that require JSON format.

**Important:** Do NOT edit this file directly. It is automatically regenerated from the YAML source when updates are needed.

## Usage Policy

### Editing Guidelines
1. **YAML is the single source of truth** — All metadata changes must be made to the YAML file first
2. **JSON is a derived mirror** — After editing YAML, regenerate the JSON file to maintain parity
3. **Never edit JSON directly** — Manual JSON edits will be overwritten during regeneration
4. **Maintain field parity** — Both files must contain identical field structure and values

### Regeneration Process
When updating metadata:
1. Edit `CANONICAL_METADATA_TEMPLATE.yaml`
2. Validate YAML syntax
3. Regenerate `CANONICAL_METADATA_TEMPLATE.json` from YAML
4. Verify JSON structure matches YAML content
5. Commit both files together

### Reusability
These templates are designed to be reused across:
- Multiple TRIZEL repositories
- Publication systems (Zenodo, DataCite, etc.)
- Archive and preservation workflows
- DOI registration processes

## Zenodo/DataCite Mapping

The metadata schema is compatible with:
- **Zenodo API** — Direct field mapping for uploads
- **DataCite Schema** — Compliant with DOI registration requirements
- **Dublin Core** — Subset mapping available

Future documentation will include explicit mapping tables for each target system.

## Provenance and Authority

- **Governance:** These templates follow TRIZEL canonical governance rules
- **Version Control:** All changes are tracked via Git
- **Cross-Repository Consistency:** Same template structure used across all TRIZEL publications
- **Non-Interpretive:** Metadata is factual and descriptive only
